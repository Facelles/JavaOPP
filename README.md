# Система управління університетом (University Management System)

Цей проєкт є консольним CLI-застосунком (Command Line Interface), написаним на Java, який імітує систему керування навчальним процесом в університеті. Програма дозволяє здійснювати повний цикл роботи зі студентами, викладачами, курсами та зарахуваннями.

## Основний функціонал (Features)

- **Студенти**: Додавання, редагування, видалення (з каскадним видаленням зарахувань), відображення, пошук (за ПІБ або email), сортування (за ПІБ) та фільтрація (за статусом чи курсом).
- **Викладачі**: Додавання, оновлення, видалення, відображення та прив'язка до навчальних курсів.
- **Курси**: Додавання, редагування, видалення, пошук курсів за кредитами або викладачем.
- **Зарахування**: Додавання студента на курс (за семестрами), виставлення оцінок (A, B, C, D, F, NA), контроль оплат, генерація транскрипту з розрахунком GPA.
- **Звіти**: Топ-N студентів за GPA, середній GPA по конкретному курсу, список студентів з неоплаченими курсами.
- **Збереження даних**: Уся база даних (студенти, викладачі, курси, зарахування) може бути збережена у `.json` файли та автоматично завантажена при наступному запуску програми (з правильним відновленням посилань у пам'яті).
- **Надійна обробка помилок**: Програма захищена від випадкового введення тексту замість чисел, не дозволяє створювати сутності з однаковими ID і перевіряє валідацію введених даних (наприклад, формат email).

## Технологічний стек

- **Мова**: Java (стандартні бібліотеки для колекцій не використовувалися за умовами ТЗ, замість них реалізовано власну логіку розширення масивів).
- **Принципи**: Об'єктно-орієнтоване програмування (інкапсуляція, поліморфізм, абстракція: класи `Person`, інтерфейс `Payable`, енуми).
- **JSON Парсинг**: [Google Gson](https://github.com/google/gson) (версія 2.10.1).

## Структура проєкту

- `src/entities/` — класи даних (`Student`, `Teacher`, `Course`, `Enrollment`, `Person`).
- `src/enums/` — перелічення (`Grade`, `StudentStatus`, `TeacherPosition`).
- `src/interfaces/` — інтерфейси (`Payable`).
- `src/services/` — сервіси для бізнес-логіки та зберігання масивів.
- `src/util/` — утиліти (`GPAUtils` для сортування та GPA, `JsonStorage` для роботи з Gson).
- `src/Main.java` — головний клас, точка входу в програму (головне меню).
- `lib/` — сторонні бібліотеки (`gson-2.10.1.jar`).

## Як запустити проєкт

### З терміналу

1. Відкрийте термінал і перейдіть до кореневої папки проєкту.
2. Скомпілюйте проєкт, підключивши бібліотеку Gson:
   ```bash
   javac -cp "lib/*" -d out src/**/*.java src/*.java
   ```
3. Запустіть програму:
   - **На macOS / Linux**:
     ```bash
     java -cp "out:lib/*" Main
     ```
   - **На Windows** (використовується крапка з комою `;`):
     ```bash
     java -cp "out;lib/*" Main
     ```

### З IntelliJ IDEA

1. Відкрийте проєкт у IntelliJ IDEA.
2. Переконайтеся, що папка `lib/` додана до залежностей (Dependencies) у вікні Project Structure.
3. Запустіть файл `Main.java` за допомогою зеленої кнопки **Run**.

## Використання програми
При першому запуску програма ініціалізується з невеликою кількістю тестових даних, щоб ви могли одразу перевірити систему звітів та меню. Виберіть пункт `6`, щоб створити JSON файли з цими даними. Зробіть потрібні зміни, закрийте програму, а при наступному запуску скористайтеся пунктом `7` для завантаження збереженого стану.
